Grant slashing allowance

Authorize a protocol contract (an enforcer) to lock and slash up to a set amount of your bond, like ERC-20 approve. Set 0 to revoke. Only ever trust audited enforcer code.

Open an obligation

As an enforcer, lock a slice of an agent's bond behind an obligation. Spends the allowance that agent granted you. Demo tip: grant your own address an allowance in My bond, then act as the enforcer here.

Set a deadline and the agent can self-release this obligation if you never resolve it, no bond can be locked forever. 0 = no expiry (only you can resolve).

Resolve an obligation

Load by ID. Release when the agent performed (bond unlocks, capacity returns). Slash on default (bond pays the creditor, capacity is burned). Only the enforcer that opened it can resolve.

How AgentBond works

1
Bond. An agent deposits USDC. The free balance is its public credit score, the bigger the bond, the more counterparties will trust it.
2
Grant. The agent approves specific protocols to lock and slash its bond, exactly like an ERC-20 allowance. Nothing can touch the bond without an explicit grant.
3
Lock → release or slash. A protocol locks a slice behind an obligation. Performed → release (capacity revolves back). Defaulted → slash (the bond pays the creditor, that capacity is burned forever).

No owner, no admin, no upgrade key, the contract only ever custodies bonds, tracked by real balance delta so it stays solvent against any token. The shared enforcement layer beneath uncollateralized agent credit, agent marketplaces, and on-chain insurance on Arc.
