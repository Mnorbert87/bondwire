1. Project Title

Arc Agentic Stack — three composable, ownerless contracts that let autonomous agents be trusted before they act and paid as they work, all settled in USDC on Circle's Arc L1.

2. Description

Every "agentic payment" demo quietly assumes two things that don't exist on-chain yet: that you can trust an agent before it touches your money, and that you can pay it continuously as it works instead of after the fact. The Arc Agentic Stack ships both as small, composable, ownerless primitives.

LayerContractAnswers
🛡️ TrustAgentBond"Can I trust this agent before it acts?" — USDC bond becomes a public, slashable credit score.
💧 SettlementStreamPay"Can I pay it per second of work, not per invoice?" — linear per-second USDC streaming, cancellable by either side.
🔥 MechanismCommitStakeV2"Who pays when the verifier lies?" — bonded verifier; on overturn the harmed party is paid its exact damage and the surplus is burned to 0x…dEaD, so collusion is never profitable.

The two primitives work standalone; CommitStakeV2 composes both into a complete trust-and-pay rail. No contract has an owner, admin, proxy or pause — immutability is the trust model.

The stack plugs into the wider Arc agent ecosystem rather than standing apart: the demo agents carry on-chain ERC-8004 identities (Arc's official registry), and an agent's bond capital can arrive cross-chain via Circle CCTP V2 before it ever posts a bond — ERC-8004 supplies identity and jobs, this stack supplies the missing economic-assurance layer.

5. Products Used

  • Circle Arc L1 (testnet, chain 5042002) — EVM L1 with USDC as native gas; deterministic finality, dollar-denominated fees.
  • USDC — both the gas asset (18-decimal native) and the value/settlement asset (6-decimal ERC-20 at 0x3600…0000). All three contracts settle exclusively in USDC.
  • CCTP V2 + Bridge Kit (Circle App Kit) — runnable cross-chain capital onboarding: 1 USDC bridged Base Sepolia → Arc and deposited into AgentBond. Arc is a native Bridge-Kit chain (ArcTestnet, CCTP domain 26); the cross-chain leg is one kit.bridge() call.
  • ERC-8004 Identity & Reputation Registry (Arc-native) — both demo agents hold on-chain identity NFTs; full surface exercised: register, ReputationRegistry.giveFeedback, IdentityRegistry.setAgentWallet.
  • Circle Wallets (Developer-Controlled) — the agent signs a full Arc lifecycle through a Circle wallet; no private key on disk touches the chain.
  • x402 over EIP-3009 — runnable demo: an agent pays an HTTP 402-gated API, settled per second on StreamPay.
  • Foundry — full adversarial test + symbolic verification (Halmos) + mutation + static-analysis (Slither) audit pipeline.

10. Product Feedback (building on Arc + Circle)

  • USDC dual-decimals is the #1 footgun. Native gas is 18-decimal while the ERC-20 is 6-decimal; mixing them silently produces amounts off by 10¹². Clear, prominent docs (or a helper that surfaces the active denomination) would save every builder this class of bug. We handle all accounting in micro-USDC against the ERC-20 and gas separately in native units.
  • USDC-as-gas is genuinely simplifying. An agent's working capital and its gas budget are the same asset — one balance to manage instead of two. This is a real ergonomic win for autonomous agent treasury logic and deserves to be a headline selling point.
  • Block timestamps are not strictly monotonic on the testnet, which matters for per-second streaming accrual; we defend against it in StreamPay. Worth documenting explicitly for anyone building time-based settlement.
  • arcscan exact-match verification worked cleanly once evm_version was pinned to cancun — a one-line note in the docs would shorten the trial-and-error.
